The directions found in this tutorial will have relevance to those using any sort of FPGA(Field Programmable Gate Array) or OS(Operating System). However, for consistent results I would advise using the following:

  • The Linux Operating System (Linux Mint or Ubuntu preferably).
  • Digilent's Nexys2 board, which uses a Xilinx Spartan3E-1200.
The Nexys2 board will make interfacing with an old keyboard and the VGA(Video Graphics Array) port easy. Ofcourse, this is not the cheapest board.

In order to use the board we will need to install additional software, the installation of which can be complicated:

  1. Xilinx WebPack ISE (the free version of ISE, the most recent version should work)
  2. Adept 2 Runtime and Utilities
These directions may change over time. Hopefully, both products can be found with a quick google search.

Installing ISE(and jumping through hoops)

The website for downloading ISE can be found at http://www.xilinx.com/support/download.html. Navigate to the ISE tab, and download the full version for Linux. This will require you to make an account with Xilinx that you will also need later for the license. This will also take half an eternity depending on the speed of your internet. The file is huge, be sure you're downloading the right one (e.g. not Vivado). Next untar the file. Installing the file is as easy as running the install script included in the file (for ISE 14.7 the name of the installation script was xsetup.)

Potential Pit Fall: The drive where you run the xsetup script needs exec privileges. (It's my drive; I'll do what I want gosh darnit!) If for some reason you are not allowed to run the script, unmount the drive where the script is located and remount with exec privileges. You might be able to use Chmod to accomplish this as well, but for me only remounting my external hard drive worked.

When installing select the options for the free Webpack software. After installing, running ISE should be as easy as sourcing the appropriate file (for me this was settings64.sh). Then use the command 'ise'.

At this point you will need a license in order to use the software. (Seemed a little silly to me since the software was free and I had to accept a few hefty User Agreements just to install it, but whatever I'm not a lawyer.) At the time of writing, licenses can be found at https://secure.xilinx.com/webreg/login.do?goto=https%3A%2F%2Fsecure.xilinx.com%3A443%2Fwebreg%2Fregister.do%3Fgroup%3Desd_oms%26tab%3DCreateLicense. In the likelihood that this link does not work, google 'Xilinx License' to find the correct site. Look for your free license (which may also be a license for Vivado, that's ok.) Load your license into ISE and you are set to make bitfiles and other cool stuff!

Installing Adept

Unlike ISE, Adept is very small. At the time of writing, Adept can be found at http://www.digilentinc.com/Products/Detail.cfm?Prod=ADEPT2. Be sure to download the correct versions of the Adept Runtime and Utilities.

Potential Pit Fall: When untarring the 'tar.gz' files pay attention to the messages displayed in the terminal. If the packages do not untar correctly, crucial files may be missing. If this happens, redownload the file, cross your fingers, and untar it again.
